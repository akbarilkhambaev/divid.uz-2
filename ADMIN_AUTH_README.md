# 🔐 Система авторизации администраторов

## 📋 Обзор

Реализована полноценная система управления администраторами для админ-панели с возможностью:
- Входа в систему
- Создания новых администраторов
- Управления ролями
- Удаления администраторов
- Защиты всех страниц админ-панели

## 🚀 Первый запуск

### Шаг 1: Создание первого администратора

При первом запуске системы перейдите по адресу:
```
http://localhost:3000/admin/setup
```

Создайте первого супер-администратора:
- **Имя пользователя**: admin
- **Email**: admin@divid.uz
- **Пароль**: ваш_безопасный_пароль

### Шаг 2: Вход в систему

После создания первого администратора вы будете перенаправлены на страницу входа:
```
http://localhost:3000/admin/login
```

Введите созданные учетные данные для входа.

## ✨ Функционал

### 1. Страница входа (`/admin/login`)
- Форма входа с полями username и password
- Валидация данных
- Проверка статуса администратора (активен/неактивен)
- Сохранение сессии в localStorage
- Автоматический редирект после успешного входа

### 2. Страница настроек (`/admin/settings`)
- Просмотр всех администраторов
- Добавление новых администраторов
- Удаление администраторов
- Статистика по администраторам
- Управление ролями (Admin / Super Admin)

### 3. Защита маршрутов
Все страницы админ-панели защищены компонентом `AdminAuthCheck`:
- Проверка наличия авторизации
- Проверка срока действия сессии (24 часа)
- Автоматический редирект на страницу входа

### 4. Верхняя панель
- Отображение информации о текущем администраторе
- Кнопка выхода из системы
- Название системы

## 🔒 Безопасность

### ⚠️ Важно для production:

1. **Хеширование паролей**
   Сейчас пароли хранятся в открытом виде. Для production необходимо:
   ```bash
   npm install bcryptjs
   ```
   И реализовать хеширование при сохранении и проверке паролей.

2. **Правила Firestore**
   Настройте правила безопасности в `firestore.rules`:
   ```javascript
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /administrators/{document} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```

3. **HTTPS**
   Используйте только HTTPS в production.

4. **Ограничение попыток входа**
   Добавьте защиту от брутфорса.

## 🎯 Роли администраторов

### Administrator (admin)
- Доступ ко всем разделам управления контентом
- Просмотр заявок
- Управление услугами, новостями, отзывами и т.д.

### Super Administrator (superadmin)
- Все права администратора
- Управление другими администраторами
- Доступ к настройкам системы

## 📝 Использование

### Добавление нового администратора

1. Войдите в систему
2. Перейдите в раздел "Настройки" в боковом меню
3. Нажмите кнопку "Добавить администратора"
4. Заполните форму:
   - Имя пользователя
   - Email
   - Роль
   - Пароль
5. Нажмите "Добавить"

### Удаление администратора

1. В разделе "Настройки" найдите нужного администратора
2. Нажмите на иконку корзины
3. Подтвердите удаление

### Выход из системы

Нажмите кнопку "Выйти" в правом верхнем углу.

## 🐛 Troubleshooting

### Забыли пароль?
Если вы забыли пароль администратора, можно:
1. Зайти в Firebase Console
2. Открыть коллекцию `administrators`
3. Изменить пароль напрямую в базе данных

### Не можете войти?
- Проверьте правильность имени пользователя и пароля
- Убедитесь, что аккаунт активен (isActive: true)
- Проверьте консоль браузера на наличие ошибок

### Сессия истекла?
Сессия действительна 24 часа. После этого необходимо войти заново.

## 🎨 Кастомизация

### Изменение времени сессии
В файле `components/admin/AdminAuthCheck.jsx` измените:
```javascript
if (hoursDiff > 24) { // 24 часа
```

### Изменение минимальной длины пароля
В файлах форм измените:
```javascript
if (formData.password.length < 6) { // минимум 6 символов
```
